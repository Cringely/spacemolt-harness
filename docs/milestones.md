# Major Milestone Tracker

The "here's the journey" view: a continuously updated report of the project's MAJOR milestones, one level above the per-flight SM-1..SM-11 record. This is the scannable timeline; the depth lives elsewhere. Each milestone links to the decision that records it (`docs/decisions.md`) and, where there's a transferable takeaway, to the lesson that teaches it (`docs/wiki/engineering-lessons.md`). Current-state detail is in `docs/STATE.md`; this page is the history STATE.md doesn't keep.

**This file is the SSOT for the milestone tracker.** The PM publishes the claude.ai Artifact ("milestone tracker") FROM this file — the Artifact is a rendered VIEW of this page, never a second source. A milestone is written here first; the Artifact is then republished to match. If the two ever disagree, this file wins and the Artifact is stale.

## How to add a milestone

A milestone earns a line here when it is a MAJOR step in the journey: a plan completed, a system going live, a whole failure class closed, a capability unlocked. Per-flight fixes and routine batches don't; they roll up into the milestone they served. When one lands:

1. Append it at the bottom (newest last) with the next `M-NN` number and its real date from STATE.md or the decisions.md entry. Don't invent dates or outcomes; cite what actually happened.
2. Keep "Delivered" to one or two plain lines; the tracker stays scannable.
3. Give it a Lesson line: the principle it taught, cross-linked to the matching `engineering-lessons.md` entry (this is the operator's teach-the-principle directive applied to the timeline). If no lesson entry fits yet, link the decisions.md entry; when the principle transfers, add a lesson and point here at it.
4. Mark status only when not yet done (`[in progress]` / `[blocked]`).
5. Republish the claude.ai Artifact from this file so the operator's remote view stays truthful (the documentation-freshness gate in AGENTS.md).

Template:

```
### M-NN · YYYY-MM-DD · Title  [status if not done]
Delivered: what shipped, in one or two lines.
Lesson: the principle it taught ([L-x](wiki/engineering-lessons.md#anchor)) / (docs/decisions.md entry).
```
